python 函数传参

1 问题引入

在第一个函数里面 a 没变，但是在第二个函数里面a变了。

原因解释

  • 当一个引用传递给函数的时候,函数自动复制一份引用,这个函数里的引用和外边的引用没有关系了.所以第一个例子里函数把引用指向了一个不可变对象,当函数返回的时候,外面的引用没丝毫感觉.而第二个例子就不一样了,函数内的引用指向的是可变对象,对它的操作就和定位了指针地址一样,在内存里进行修改。

  • 关于可变对象和不可变对象：对象有两种,“可更改”（mutable）与“不可更改”（immutable）对象。在python中，strings, tuples, 和numbers（int, float bool）是不可更改的对象，而 list, dict, set 等则是可以修改的对象。(这就是这个问题的重点)

2 新的例子

在jupyter notebook 或者在交互式命令行中运行（每单独一行视为一个代码块）

出现 a 和 b 指向不同的内存地址。

原因解释

  • python中是有小整数对象池和大整数对象池概念的， Python 对小整数的定义是 [-5, 256] 这些整数对象是提前建立好的，不会被垃圾回收。(小整数缓存池，大整数缓冲池),之前第一个例子里面的1 2 都属于小整数池中，直接拿节省时间，但是计算机不可能存那多多整数，所以只存了小整数池中的数字。

  • 上面的例子中 a b 属于不同的代码块，因此是不同的对象。（放在一个py文件中再运行的话结果会不同，参考后面新的例子解释）

  • python有个intern机制，简单说就是维护一个字典，这个字典维护已经创建字符串(key)和它的字符串对象的地址(value),每次创建字符串对象都会和这个字典比较,没有就创建，重复了就用指针进行引用就可以了。

    • 1.小整数 [-5, 256] 共用对象，常驻内存
    • 2.单个字符共用对象，常驻内存
    • 3.单个单词，不可修改，默认开启intern机制，共用对象，引用计数为0，则销毁
    • 4.字符串（含有空格），不可修改，没开启intern机制，不共用对象，引用计数为0，销毁

    3 新的例子

A 中 a 和 b 的地址又变得相同了，但是与B中的不一样。

解释

  • 大整数池的原因：如果处于相同代码块(类中或者在同一个py文件中)的大整数对象，它们是同一个内存地址（同一对象）， 会让其共享引用， 不会在开辟一个内层空间存放，目的也是为了优化python内部的性能。

4 扩展

现象：某一地址出现了好多次。
原因： 当一个对象引用计数为0时，会调用tp_dealloc的函数，这个通常是将对象所占用的内存还给python解释器，一般是还给python中的对象内存池，等待新生。而不是将空间还给操作系统。
